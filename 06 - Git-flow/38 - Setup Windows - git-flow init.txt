Git-flow = é uma extensão de bibliotecas do git

git flow 
usage: git flow <subcommand>

Available subcommands are:
   init      Initialize a new git repo with support for the branching model.
   feature   Manage your feature branches.
   bugfix    Manage your bugfix branches.
   release   Manage your release branches.
   hotfix    Manage your hotfix branches.
   support   Manage your support branches.
   version   Shows version information.
   config    Manage your git-flow configuration.
   log       Show log deviating from base branch.

Try 'git flow <subcommand> help' for details.

git flow init = inicia nosso repositorio no modo git flow

Which branch should be used for bringing forth production releases?
   - develop
   - master
Branch name for production releases: [master]
obs.: pede para selecionar uma branch q será usado como produção de releases
branch = master


Which branch should be used for integration of the "next release"?
   - develop
Branch name for "next release" development: [develop]
obs.: pede para selecionar uma branch q precisa ter o código para o candidato em produção, é a branch onde vamos programando e selecionando nossas features
branch = develop


How to name your supporting branch prefixes?
Feature branches? [feature/]
obs.: vai pedir o nome do prefixo de suas branches feature
ex.: feature/(branch)

Bugfix branches? [bugfix/]
obs.: vai pedir o nome do prefixo de suas branches de bugfix, onde vai vai cpnsertar algum erro
ex.: bugfix/(branch)

Release branches? [release/]
obs.: vai pedir o nome do prefixo de suas branches de release, as candidatas para ir a produção
ex.: release/(branch)

Hotfix branches? [hotfix/]
obs.: vai pedir o nome do prefixo de suas branches de hotfix
ex.: hotfix/(branch)

Support branches? [support/]
obs.: vai pedir o nome do prefixo de suas branches de suporte
ex.: support/(branch)

Version tag prefix? []
obs.: vai pedir o nome do prefixo de suas tags 
ex.: /(branch)
